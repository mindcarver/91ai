# 机器学习教程文档规则

适用于 `docs/machine-learning/` 下的系列教程文章。

## 标准章节结构

stage0-stage4 的教程文章应包含以下 h2 章节：

1. `## 为什么这很重要`
2. `## 核心概念`
3. `## 工作原理（简化的心智模型）`
4. `## 工作原理（详细机制）`
5. `## 代码示例`
6. `## 真实案例`
7. `## 权衡取舍以及何时不该使用`
8. `## 关键要点`
9. `## 延伸阅读`

stage5+ 的文章允许适当调整章节，但核心部分（为什么重要、工作原理、代码示例、关键要点）不应省略。

## 格式要求

- 文件开头可以有 HTML 注释块记录研究来源，然后是 `# 标题`
- 代码示例使用 Python，放在 ` ```python ` 围栏内
- 确保代码块正确闭合（反引号围栏成对）
- 避免在代码块中出现尾部空白行

## 文件组织

- 按 stage 子目录分阶段：stage0-ml-basics 到 stage8-advanced
- 文件名格式：`01-topic-name.md`，数字前缀表示顺序
- 每个目录不需要 README.md
