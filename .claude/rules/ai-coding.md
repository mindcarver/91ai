# AI Coding 工具文档规则

适用于 `docs/ai-coding/` 下的工具评测文件（如 `claude-code.md`、`cursor.md`）。

## 必须包含的章节

每个工具文件必须包含以下 h2 章节（顺序可调整）：

- `## 推荐阅读顺序`
- `## 官方资料`
- `## 学习资料`
- `## 生态项目与能带来的东西`
- `## 使用场景分类`
- `## 评测维度建议` 或等效的评测章节
- `## 当前判断`

## 格式要求

- 文件开头：`# 工具名` + 空行 + `> 更新日期：YYYY/MM 或简要描述`
- 列表统一使用 `-`，不用 `*`
- 外部链接用 `[显示文字](URL)` 格式
- 避免营销话术，说明适合谁、提供什么、有什么限制

## 系列文章索引同步

修改 `docs/ai-coding/` 下的系列文章（新增、删除、重命名、调整编号）时，必须同步更新以下索引：

### 需要检查的索引位置

1. **`docs/ai-coding/README.md`** — 对应系列的表格或列表条目。确认系列名称、文章数量、链接路径准确。
2. **`docs/README.md`** — 如果系列是新增目录或影响顶级导航结构，更新对应的索引入口。
3. **`README.md` badge** — 如果变更影响项目计数，运行 `./scripts/check.sh badges` 验证。

### 检查流程

每次修改系列文章后，按以下顺序操作：

1. 确认目标目录下的文件列表（`ls docs/ai-coding/<series-dir>/`）
2. 对比索引文件中的条目与实际文件是否一致
3. 更新索引中的文章数量、链接路径、描述文字
4. 运行 `./scripts/check.sh links` 验证内部链接

### 不需要更新索引的情况

- 仅修改文章内容（不改文件名、不增删文件）
- 仅修改格式或修复错别字
