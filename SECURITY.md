# 安全策略

91ai 是文档与知识库项目，没有独立运行时服务。安全风险主要来自收录内容、外部依赖和贡献过程。

## 关注范围

- 指向恶意、仿冒、被接管或会触发危险下载的链接。
- 文档、提示词或示例中隐藏的提示注入和诱导执行内容。
- 脚本、Skill、MCP Server、GitHub Actions 或安装命令的供应链风险。
- 可能泄露密钥、令牌、生产凭据、私有 URL 或内部信息的内容。
- 会让读者在未充分知情时授予高权限或执行破坏性操作的说明。

普通错别字、非敏感失效链接和内容时效问题请使用 [内容问题表单](https://github.com/mindcarver/91ai/issues/new?template=content-report.yml)。

## 私下报告

如果问题包含敏感信息或可利用细节，请不要创建公开 Issue。当前仓库尚未启用 GitHub Private Vulnerability Reporting（核验日期：2026-07-10）；请通过根 [README](README.md) 公开的微信 `mindcarver` 私下联系维护者。请先只发送问题概要，不要在未确认接收方式前发送真实密钥、个人数据或生产凭据。

报告中请尽量包含：

- 受影响的文件、链接或内容位置。
- 风险说明、复现条件和可能影响。
- 安全的验证证据；请移除真实密钥和个人信息。
- 建议的修复方式或可信替代来源（如有）。

维护者会在可用时间内确认报告、核实影响并协调修复。修复完成前，请不要公开可利用细节。

## 安全贡献要求

- 使用占位符代替真实凭据和内部地址。
- 优先链接官方仓库、官方文档和可核验发布页。
- 不直接运行来源不明的安装脚本、Skill 或 MCP Server。
- 对需要 shell、文件系统、浏览器、GitHub 或云资源权限的工具明确标注权限边界。
- 发现外部资源被接管或内容变质时，先移除或隔离链接，再讨论替代方案。

本策略适用于默认分支上的当前内容；历史版本可能不再维护。
