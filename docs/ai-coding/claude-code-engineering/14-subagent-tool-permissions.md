# 工具权限：只读审计代理为什么不能有写权限

**TL;DR：** Subagent 的工具权限应该按任务最小化。审计、研究、测试诊断通常不需要写文件权限。

## 问题

如果所有 subagent 都继承主会话全部工具，隔离上下文的收益会被权限风险抵消。一个只负责审查的代理不应该能改代码，一个只负责研究的代理不应该能运行部署脚本。

## 权限分级

| 任务 | 推荐权限 |
| --- | --- |
| 代码探索 | Read、Grep、Glob |
| 代码审查 | Read、Grep、Glob |
| 测试诊断 | Read、Grep、Glob、Bash 测试命令 |
| 实现任务 | Edit、Write、Bash，但限制目录和命令 |
| 安全审计 | 只读优先 |

## 落地练习

把 reviewer 的工具限制成只读，然后让它审查一段明显有问题的代码。它应该输出问题和建议，而不是直接修改。

## 权衡

权限太窄会让 subagent 无法完成任务。解决方法不是一开始给全权限，而是按失败证据逐步增加工具。
