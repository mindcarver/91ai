# 安装、登录、权限模式和第一个真实仓库任务

**TL;DR：** 第一次使用 Claude Code，不要从大重构开始。先验证登录、权限、命令执行、文件修改和测试闭环是否正常。

## 问题

很多团队试用 AI Coding 工具时，第一天就交给它复杂需求。失败后结论是“AI 不靠谱”。更合理的方法是先做运行环境体检：Claude Code 能不能进入仓库、读懂文件、执行命令、改一处小问题并验证结果。

## 核心机制

第一次仓库任务应覆盖五件事：

1. 当前目录是否正确。
2. 项目说明是否足够。
3. 权限模式是否允许必要工具。
4. 测试命令是否能运行。
5. Claude Code 是否会报告未验证风险。

## 落地练习

选一个非核心 bug 或文档更新任务，按这个提示执行：

```text
先阅读 README、package.json 和测试说明。
告诉我这个项目如何安装、运行、测试。
然后修复 docs 中一个明显过期的命令。
修改后运行最小相关检查。
最后报告 diff、验证结果和剩余风险。
```

## 验收标准

- 它没有直接改大量文件。
- 它能说明为什么读这些文件。
- 它能运行或解释无法运行的检查。
- 它能列出剩余风险。

## 权衡

权限越开放，试用越顺滑，但风险越高。第一天可以允许读写普通源码，但不要允许生产凭据、部署命令和数据库写操作。
