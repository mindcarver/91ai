# PostToolUse / Stop：自动格式化、测试和结果记录

**TL;DR：** 修改完成后的验证比修改本身更重要。`PostToolUse` 和 `Stop` 可以帮你自动提醒测试、记录结果和形成审计轨迹。

## 问题

Claude Code 修改文件后，如果没有验证，很难判断结果是否可靠。人类开发也会忘记跑测试，AI 更需要流程提醒。

## 适合做什么

### PostToolUse

- 文件修改后提示运行相关格式化。
- 测试失败后收集错误摘要。
- 编辑关键文件后记录变更。

### Stop

- 总结本轮修改。
- 检查是否运行验证。
- 提醒未完成测试。
- 写入会话审计日志。

## 落地练习

给仓库定义“验证报告”格式：

```md
## Verification
- Commands run:
- Passed:
- Failed:
- Not run:
- Residual risk:
```

要求 Stop Hook 或团队规则每轮结束都输出这段。

## 权衡

不要让 Hook 自动跑全量测试。大仓库全量测试很慢，容易导致体验崩。先提醒最小相关测试。
