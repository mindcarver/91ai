# 数据库、监控和设计系统：高价值 MCP 场景

**TL;DR：** MCP 的高价值场景不是“能接多少工具”，而是接入那些会显著减少上下文搬运的系统：数据库、监控、设计系统和项目管理。

## 问题

AI 需要真实状态才能做工程判断。代码只是状态的一部分，另外还有用户反馈、错误日志、指标、设计稿和数据模型。

## 高价值场景

### 监控系统

用于分析线上错误、慢查询、异常指标。适合只读。

### 数据库

用于理解 schema 和数据分布。第一阶段只允许查询元数据或只读样例。

### 设计系统

用于读取组件规范、设计 token、页面结构。

### 项目管理

用于同步需求、验收标准和任务状态。

## 落地练习

挑一个系统接 MCP，并写一张权限表：

| 工具 | 允许 | 禁止 |
| --- | --- | --- |
| Sentry | read issue, read event | delete, resolve |
| DB | read schema | update, delete |
| Figma | read design | publish |

## 权衡

越接近生产数据，越要保守。MCP 接入顺序应该是：文档和 issue -> 监控只读 -> 数据库只读 -> 写操作。
