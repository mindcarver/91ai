# 三类高价值 Subagent：探索、审查、测试

**TL;DR：** 最值得先做的 Subagent 是 explorer、reviewer、test-runner。它们任务边界清楚，能独立工作，结果容易验证。

## 问题

团队一开始常创建很多花哨角色：架构师、产品经理、性能专家、文档专家。数量变多后，反而不知道什么时候该用哪个。

## 三个起点

### Explorer

只读探索代码库，回答“相关文件在哪、调用链是什么、已有模式是什么”。

### Reviewer

审查当前 diff，重点找 bug、缺测试、安全风险和边界条件。

### Test Runner

运行测试、归纳失败、定位最小可复现路径。

## 落地练习

先创建这三个 subagent，每个只写 10-20 行说明。用同一个 bug 修复任务测试：

1. Explorer 找相关文件。
2. 主会话实现。
3. Test Runner 跑测试。
4. Reviewer 看 diff。

## 权衡

角色越多，路由越难。先把三个基础角色跑稳定，再扩展性能、安全、文档等专门角色。
