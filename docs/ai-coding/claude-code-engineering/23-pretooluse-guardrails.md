# PreToolUse：阻断危险命令和高风险文件写入

**TL;DR：** `PreToolUse` 适合做工具调用前的门禁。它应该拦截少数明确危险的操作，而不是承担复杂业务判断。

## 问题

Claude Code 能运行命令和写文件。高风险操作包括删除文件、修改 secrets、运行部署、写数据库、推送代码。仅靠提示词不够。

## 适合阻断的操作

- `rm -rf`。
- 修改 `.env`、secret、证书文件。
- 直接运行生产部署命令。
- 对生产数据库执行写操作。
- 在外部 PR 上读取 secrets。

## 设计原则

阻断规则应该：

- 明确。
- 可解释。
- 可覆盖。
- 有日志。
- 不依赖模型判断。

## 落地练习

写一份高风险文件清单：

```txt
.env
.env.*
*.pem
*.key
infra/prod/**
migrations/**
```

把这些路径接入 PreToolUse 策略。第一次只提示，不阻断；观察一周后再升级部分规则为阻断。

## 权衡

误拦截会破坏开发体验。先记录，再提示，最后阻断，是更稳的落地顺序。
