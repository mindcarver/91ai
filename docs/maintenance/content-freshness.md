# 内容保鲜规则

91ai 同时包含快速变化的产品信息和相对稳定的工程方法。不同内容不能用同一个“最新”标准，维护时先判断时效等级。

## 时效等级

| 等级 | 典型内容 | 维护要求 |
| --- | --- | --- |
| 高 | 价格、版本、模型列表、API、安全行为、许可证、迁移步骤 | 写明 `最后核验：YYYY-MM-DD`，修改时回到官方来源重新核验 |
| 中 | 工具对比、项目数量、生态排名、工程实现细节 | 标注资料或实测日期，不把快照表述成长期结论 |
| 低 | 基础原理、通用设计模式、方法论 | 来源或核心前提变化时更新 |

## 状态标记

- **当前有效**：有可核验来源，且没有发现与当前版本冲突。
- **历史快照**：结论只对特定日期、版本、环境或评测协议成立。
- **待复核**：来源失效、版本不明或新证据与原结论冲突。保留内容时必须明确显示这个状态。

无法核验时，不用“大概仍然有效”掩盖不确定性；应改为历史快照、标记待复核，或移除无法支撑的当前式结论。

## 维护流程

1. 使用 [内容问题表单](https://github.com/mindcarver/91ai/issues/new?template=content-report.yml) 报告过时、失效或冲突内容。
2. 修复时记录官方来源、最后核验日期和受影响的版本。
3. 保留有历史价值的旧结论时，把它降级为“历史快照”，不与当前建议混写。
4. 运行 `./scripts/check.sh all`，确认格式、内部链接和计数没有漂移。

当前这是人工维护规则，不代表仓库已自动验证每条内容的时效性。
