## Why

AI 资料已经足够多，真正缺的是判断力。

这个项目关注：

- 哪些 AI 工具真的值得技术人投入时间使用
- 哪些资料值得系统学习
- 哪些方案适合真实项目落地
- 哪些 Demo 很漂亮，但不适合生产环境

> 别人给你一堆链接，这里帮你建立 AI 技术判断力。

## Evaluation

每个工具会尽量从真实工程角度评估，而不是只看官网 Demo 或 GitHub Star。

<table>
<tr><td><strong>维度</strong></td><td><strong>看什么</strong></td></tr>
<tr><td>使用价值</td><td>是否解决真实、高频、重要的问题</td></tr>
<tr><td>上手成本</td><td>安装、配置、文档、学习曲线</td></tr>
<tr><td>工程集成</td><td>是否容易接入真实项目和现有工作流</td></tr>
<tr><td>生产可用性</td><td>稳定性、权限、日志、监控、回滚</td></tr>
<tr><td>可控性</td><td>是否容易调试、约束行为、定位问题</td></tr>
<tr><td>成本</td><td>订阅成本、API 成本、部署成本、迁移成本</td></tr>
<tr><td>风险</td><td>数据安全、供应商锁定、不可解释行为</td></tr>
</table>

## Principles

- 资料要全，但不只追求数量。
- 评测要明确，不写模糊的万能推荐。
- 优先真实使用体验，谨慎引用营销话术。
- 关注工程落地、长期维护、安全边界和成本。
- 过时、低质量或失效资料会被移除或标记。
